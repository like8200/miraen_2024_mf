CSS style

css style 규칙

  • 공통 선택자(*)는 웹페이지의 성능 저하를 가져오므로 사용하지 않는다.
  • !important는 불가피한 상황을 제외하고는 사용하지 않는다.
  • 인라인 스타일의 우선 순위가 높으므로 인라인 스타일는 지양한다.
  • 페이지에 종속되는 스타일의 사용을 지양한다.
  • 모든 속성은 영문 소문자로만 작성한다.
  • 기본 요소에 box-sizing 속성값을 지정하지 않는다.
  • 컬러값은 단축 가능하다면 단축한 값으로 기재한다. (ex. #fff)
  • 선택자를 다수 작성할수록 우선 순위가 높아지므로 선택자는 가능하다면 최소 단위 / 루트 단위로 진행한다.

SCSS 작성 규칙

  • 파일 구조는 가능하다면 컴포넌트 형식으로 파일을 나누어 작업한다.
  • 검색에 용이하도록 SCSS 파일명과 해당 파일 최상위 클래스명은 동일하게 사용한다. (ex: _page-header.scss 내부 사용된 클래스명은 .page-header)
  • 예외적인 상황이 아닌 경우, 페이지 내에서 개별적으로 .scss , .css 파일을 호출하지 않는다. 하나의 스타일시트에 임포트하는 방식으로 진행한다.
  • 자기 자신을 가르치는게 아닌 경우, 네스팅 사용시 &를 사용하지 않는다. & 클래스명 검색돼지 않는 문제가 있다.
  • 종료 주석은 작성하지 않는다.
  • 코드 그룹 간 1줄의 빈 줄을 허용한다. 단, 1줄을 초과하지 않는다.
  • z-index 10 이상 사용시 variables에 추가해서 영향 확인 후 사용 한다.
