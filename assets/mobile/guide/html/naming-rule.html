Naming

Name 명명규칙 (ID / Class / Component)

  • ID는 개발을 위해서만 사용하고, 퍼블리싱 작업시에는 앵커 및 label 등의 연결하는 상황 등을 제외하고는 사용하지 않는다.
  • 불가피한 경우를 제외하고 클래스명은 직관적으로 짓는다.
  • 예약어를 단독으로 1차 Class selector로 사용하지 않는다. (ex : desc, title 만 단독으로 root 하위 클래스로 사용 불가)
  • 서로 약속된 예약어만 약식으로 사용한다.
  • Class Selector 표기는 kebab-case를 사용한다.(ex: page-header)
  • File name(JS/SCSS) , Component name은 PascalCase를 사용한다.(ex:PageHeader.js)
  • 체인 선택자의 개수는 3개까지 권장한다.
  • 클래스명에 넘버링은 최대한 지양한다.
  • 클래스명에 컬러나 위치를 표현한 명칭 사용을 지양한다.
  • BEM방법론 명칭을 사용하지 않는다.

예약어

구분 예약어 설명
바깥 영역 -s 동일한 객체의 그룹핑 역활
바깥 영역 -wrap 동일 객체가 아닌 - UI 영역 단위로 그룹화
안쪽 영역 -inner 내부 영역
머리글 영역 -header 전체 영역의 목차 포함
사이드 -aside 사이드 영역
본문 영역 -contents 전체 영역의 본문
바닥글 영역 -footer 전체 영역의 바닥글
제목 영역 -title 전체 영역의 제목
네비게이션 -nav 탐색 역할을 갖는 범위의 메뉴
활성화 active 객체의 상태가 변경될 경우 활성화된 상태
비활성화 disabled 객체의 상태가 변경될 경우 비활성화된 상태
